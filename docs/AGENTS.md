# AGENTS.md — Design and production Vault

These instructions extend the repository root guidance for `docs/`.

- Do not duplicate current runtime values from `game_data/`; document intent, ranges, and rationale.
- Accepted notes must contain no unresolved `TBD`.
- TASK and ADR frontmatter must pass governance checks.
- Update source-of-truth notes, linked specs, tests/fixtures, and TASK evidence in the same change when behavior changes.
- Chat summaries are not source of truth.
- Do not upgrade E0–E2 evidence into a fun/balance claim requiring E3–E5.
- Keep Obsidian wikilinks valid and avoid machine-private paths.
