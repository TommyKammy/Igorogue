# Development Plan

## 運用

Obsidianを制御盤、Gitを実装履歴、`game_data/`をランタイム値の正本、Codexを承認済みタスクの実行者とする。

## 原則

- 一タスク一成果。
- 一Codexスレッド一TASK。
- 実装と独立レビューを分ける。
- 「実装済み」「テスト済み」「面白さ検証済み」を別状態にする。
- M3 Acceleration Labが成立する前にコンテンツ量産しない。
