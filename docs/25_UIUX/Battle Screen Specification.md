# Battle Screen Specification

![[mockup_01_battle_screen.png]]

## レイアウト

- 中央：7×7盤。最大の視覚面積。
- 左：気、魂、予測黒領地収入、全流儀共通の余勢0〜2、王石安全度。
- 右上：敵名、次意図、対象、反攻ゲージ／100、自然増加、Pending badge。
- 右中：遺物5枠。
- 右下：選択手の差分予測。
- 下：手札、山札、捨て札、ターン終了。
- 上：Act、戦闘、ターン、妙手倍率、コミ。

## 盤面表示

- 盤面の視覚方向は[[Coordinate System and Initial Position]]に従い、左下が`(1,1)`、右上が`(7,7)`。
- 座標ラベルは通常時に省略できるが、チュートリアル、デバッグ、アクセシビリティ設定で表示可能にする。
- カーソル／ツールチップへ表示する点はCanonicalPoint `(x,y)`。

- 黒領地は翡翠の薄いハッチ。
- 白領地は赤紫の薄いハッチ。
- 中立は無着色。
- アタリは石周囲の脈動線。
- 即捕獲可能点は剣型カーソル。
- 自王石即死点は赤い禁止枠。

## 施設表示

- 施設は交点を覆う石アイコンではなく、空点カーソルを残す下層glyphとして描画する。
- activeは通常色、disabledは低彩度＋停止アイコン。
- 石カード選択時、施設点も合法点として明るくできる。
- 合法配置で施設を失う点には`施設を破壊`警告を差分予測へ出す。
- 不合法点では施設破壊を予告しない。
- 施設ツールチップは「空点として扱う／石が置かれると破壊」を常時確認できる。


## 余勢表示

- 余勢は特定領地や流儀へ紐付けず、左resource railに`0/2`〜`2/2`で表示する。
- eligibleな`frontline`石札を選ぶと、通常合法点と`momentum_reach`追加点を異なるglyphで示す。
- 余勢点のhoverでは、起点石、中間空点、余勢`1消費`、解決後予測値を表示する。
- 通常合法でもある点には余勢glyphを出さず、余勢を消費しない。
- 前線前進ドローが発生する点には1ドローiconを表示する。
- 不合法点では気・余勢消費を予告しない。

## カード選択時

合法点だけを明るくする。ホバー点で以下を表示する。

```text
白1群をアタリ
新領地3目
余勢+1
妙手 ×2.8 → ×3.1
反攻 +8
自グループ実呼吸点 2
```

## 入力

- 左クリック／A：選択・決定
- 右クリック／B：戻る
- Space／Y：ターン終了
- Tab／肩：盤面情報レイヤー切替
- 数字1〜7：手札選択


## 反攻表示

- barは200 internal unitsを100表示点へ変換する。`.5`は必要時だけ数値ラベルへ表示する。
- 敵ターン終了の自然増加を`次 +6.5`のように常時表示する。
- 選択手が熱、過伸展、犠牲batchを生む場合、差分予測へreason付きで表示する。
- 閾値到達時はbarを0へ戻すだけでなく、`反攻準備完了：次の敵ターン +1手`badgeを残す。
- Pending中のbarは次回以降へ持ち越すoverflowを表す。
- 妙手x3へ達して熱が発生した直後も、残りのプレイヤー操作を妨げない。
