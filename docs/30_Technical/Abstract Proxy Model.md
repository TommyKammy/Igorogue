# Abstract Proxy Model

## 目的

会話中の設計検証で使った抽象モデルの考え方を、再実行可能な参考実装として保存する。`tools/abstract_sim/`に標準ライブラリのみのPython実装がある。

## 抽象状態

- economy：領地と施設の成長力
- setup：遅延準備と形作り
- attack：王石への変換力
- defense：敵圧への耐性
- draw：手札回転
- conversion：経済・準備を攻撃へ変える能力
- catalyst：発火札・遺物へのアクセス
- volatility：外れ値の大きさ
- engine：一戦中の蓄積
- pressure：敗北圧
- heat：高コミ爆発の反動

## モデル化していないもの

- 49交点の具体配置
- グループ共有呼吸点
- 切断点、二眼、死活
- カードの対象合法性
- 実際の敵意図候補

## 利用方法

1. 100案を粗く比較。
2. 候補を正式盤面シミュレーターへ。
3. 人間プレイテストで判断。

## 現在の参考ベースライン

再構築モデルのmatrixでは、全スタイル・印構成を均等にした平均で、熟練方策は概ね以下となった。

- 完走率：約77%
- 真の爆発：約58%
- 早期爆発：約23%
- 事前仕込みを伴う爆発：約98%
- 爆発なし勝利：約32%
- 爆発なし敗北：約10%

これらはコード調整例であり、製品目標の証拠ではない。詳細は[[SIM-0001 Reference Proxy Baseline]]。
